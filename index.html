Project — Boss Fight System

Overview

This project focuses on building a dynamic boss with quadrant-based movement, multiple projectile patterns (360 burst, rotating bullets, wave projectiles) and player-controlled projectiles including a laser weapon pickup. The boss reacts differently depending on its quadrant and executes context-sensitive moves.

Problem solved

Many student projects have static enemies with predictable behavior. This project explores richer AI behavior, player-item interaction (laser pickup) and patterns that encourage skillful play — including destructible projectile waves and dash telegraphing.

Unique features

  • Quadrant-based boss movement and move selection
  • 360-degree burst and rotating bullet patterns
  • Projectile wave that can be destroyed by player's attack
  • Player pickup: Laser gun with directional firing

Project Development — Progress Log

Timeline & Key Milestones

  • Implemented real-time combat and slash attacks.
  • Built projectile system and added laser gun pickup (late Sep — Oct).
  • Developed boss quadrant movement, 360 burst and rotating patterns (Oct 20–31).
  • Polished sprites, fixed dash bug where boss locked to player coordinates (Nov 20–21).

Challenges & Solutions

Examples: A projectile direction bug was solved by switching movement logic to use switch-cases for player-facing orientation. Dialogue color "undefined" errors were handled by adding defensive checks. A 360 spread issue was resolved with AI-assisted debugging and iterative testing.
